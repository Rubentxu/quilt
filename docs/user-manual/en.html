Quilt combines a structured outliner, a knowledge graph, and AI agents into a single local-first workspace. This guide takes you from zero to productive.

01 — Getting started

Installation and startup

One command to spin up the entire development environment.

just dev
ℹ

This starts the API backend at localhost:3737, the React frontend at localhost:5173, and the WASM watcher for graph operations.

On first run, Quilt generates an API key and prints it to the server console. Create it in quilt-ui/.env:

VITE_QUILT_API_KEY=4eb04d25-ba93-44e3-baec-9c21ae5a9a2c

04 — Blocks

Slash commands — /

Transform any block. Type / at the start to open the menu.

Slash command menu with categories
CommandProperties set
/h1, /h2, /h3level:: 1 | 2 | 3
/codeblockType:: code
/quoteblockType:: quote
/tasktype:: task + status:: todo
/querytype:: query + dsl:: (prompts for DSL)
/cardcard-shape:: (prompts for shape)
/dividerVisual separator
/bullet, /numberedBullet or numbered list
✓

Semantic roles vs. visual types: /task, /query, and /card set roles (properties like type::). The other commands (/h1, /code, etc.) set visual formatting or block type (blockType::).

08 — Navigation

Block zoom — ?zoom=$blockId

Deep-link to a specific block. Its content becomes the focus of the view.

/journal/2026-06-07?zoom=abc123-def456

This opens the journal focused on block abc123, with its content expanded and the rest of the page dimmed. Ideal for sharing precise block references within a page.

09 — Dates

Natural language dates

In property values you can write dates as plain text.

You writeResolves to
due:: todayThe current date
due:: tomorrowTomorrow's date
due:: yesterdayYesterday's date
⚠

Natural dates are resolved in the frontend at render time. The value stored in the database is the original string (today, tomorrow, yesterday), not the computed date.

10 — AI

Agent blocks — type:: agent-run

External agent executions recorded as blocks.

A block with type:: agent-run logs an agent execution. Properties:

PropertyDescription
agent::Agent name
model::Model used
run-status::Status: Queued / Running / Completed / Failed / Cancelled
started-at::Start timestamp
completed-at::End timestamp
summary::Summary of what was done

Status colors in the UI:

StatusColor
Queued / CancelledGrey
RunningBlue (info)
CompletedGreen (success)
FailedRed (danger)
11 — Views

Saved views — type:: view

Composition over inheritance: multiple views over the same query.

A saved view is a type:: view block that references an existing query block via data-source::.

ℹ

Composition pattern: the query block contains the DSL. The view block defines the renderer and configuration (group-by, sort). Multiple views can share the same query.

PropertyTypeDescription
view-type::selecttable | kanban | calendar | list | graph | cards | timeline
data-source::block-refUUID of the query block
view-name::stringHuman-readable name
view-icon::stringLucide icon name
group-by::property-keyGrouping property
sort::JSONSort configuration
