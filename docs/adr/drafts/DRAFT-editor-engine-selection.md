# DRAFT ADR: Editor Engine Selection — Outliner and Code Blocks

## Context

Quilt's frontend is migrating from Leptos to React (see DRAFT-hybrid-wasm-boundary). The current editor engine is CodeMirror 6, integrated via manual wasm-bindgen FFI and mandated by ADR-0007. In the new React context, native JS editor libraries become viable, and the structural requirements of Quilt's outliner UX must be re-evaluated.

Quilt requires:
- Block-based editing (each block is an independent editable unit)
- Tree navigation (indent/outdent, collapse/expand, reorder)
- Inline formatting (bold, italic, links, tags, properties)
- Slash commands (/menu with block type selection)
- Drag-and-drop block reordering
- Virtual scrolling for large documents
- Embedded code blocks with syntax highlighting
- Bidirectional references and backlinks display

ADR-0007 was written under Leptos + wasm-bindgen constraints that no longer apply. The React migration is the context change that triggers this re-evaluation.

## Decision

**Tiptap (ProseMirror + React) is selected as Quilt's primary outliner editor.**

**CodeMirror 6 is retained for embedded code blocks only** (syntax highlighting, code editing within blocks).

**Lexical (Meta) is the fallback/second choice** if Tiptap proves unsuitable during POC validation.

**Slate is excluded** due to maintenance risk (long gaps between releases, documentation lag).

| Library | Role | Rationale |
|---------|------|-----------|
| **Tiptap** (ProseMirror) | Primary outliner | Native block model (Node-based), tree operations built-in (indent/outdent/lift/sink), first-class React integration (`@tiptap/react`), production outliner usage (Linear, GitLab), active ecosystem |
| **CodeMirror 6** | Embedded code blocks | Unmatched for code editing, syntax highlighting, and language support. No longer the primary editor. |
| **Lexical** | Fallback candidate | Powerful custom block support, Meta-backed. Higher complexity than Tiptap for outliner use case. |

## Why

1. **Structural fit over path dependency**: CodeMirror 6 is a code editor — linear document model, no native block structure, no tree operations, no block-level drag-drop. Building an outliner on CM6 requires building outliner primitives from scratch on a foundation designed for a different problem. "We already use it" is a sunk-cost argument, not an architectural one.

2. **ProseMirror maps to Quilt's block model**: Tiptap's underlying ProseMirror schema uses a Node-based document model that maps 1:1 to Quilt's block structure. Tree operations (indent, outdent, lift, sink) are built-in, not custom-built.

3. **ADR-0007 assumptions have changed**: ADR-0007 was written when Leptos + wasm-bindgen constrained editor library choices to those with Rust FFI. In React, native JS libraries are directly usable — including libraries designed specifically for block-based editing.

4. **CM6 retained where it excels**: CM6 is the best tool for code editing within blocks. Retaining it for embedded code blocks preserves existing integration knowledge where it's structurally appropriate.

## Consequences

- **ADR-0007**: Revised. CM6 role changes from "primary editor engine" to "code block editor." Tiptap added as primary outliner.
- **New dependency**: `@tiptap/react`, `@tiptap/core`, `@tiptap/extension-*` packages. Bundle size impact must be measured.
- **Migration cost**: ~300 lines of wasm-bindgen CM6 FFI removed. New Tiptap integration written in TypeScript.
- **Custom extensions**: ProseMirror extensions likely needed for Quilt-specific block types (properties, references, journals).
- **POC required**: Block-based editing with Tiptap for indent/outdent, slash commands, and inline formatting with Quilt block types.
