Quilt — AI-first Knowledge Graph

User Manual & Reference — v0.1.0

1. Overview

Quilt is an AI-first knowledge graph built in Rust with a React frontend. It inherits the Logseq block-outliner model and reimagines it with a clean hexagonal architecture, first-class MCP protocol support, and graph algorithms powered by petgraph.

Key principles:

  • MCP-first — all operations are MCP tools. AI agents are first-class users.
  • Type safety — properties are typed, not strings in frontmatter.
  • Clean architecture — Domain → Application → Infrastructure → Presentation.
  • Graph-native — petgraph powers 8 graph analysis tools.
  • WASM-compatible — quilt-core compiles to wasm32-unknown-unknown.
StackRust 2024, SQLite, Tokio, Axum, React 19, TypeScript, TipTap 2, Vite
Server port3737

2. Getting Started

Prerequisites

Rust 1.85+ | Node 20+ | just (optional)

Quick Start

# Clone and run
git clone <repo> quilt
cd quilt
just dev

A single command starts the full dev environment:

ComponentPortWhat
Rust server3737API + WebSocket + serves SPA
Vite dev5173React SPA with HMR
WASM watcher—Rebuilds quilt-core on edit
Node asset watcher—Syncs prod bundle to wasm_assets/

Auth

The server auto-generates a UUID v4 API key on first start and prints it to the console. Copy it to quilt-ui/.env as VITE_QUILT_API_KEY=<key>. All API calls require Authorization: Bearer <key>.

4. Core Concepts

Blocks

The fundamental unit. Every piece of content is a block. Blocks form a tree via parent_id. Properties are typed key-value pairs stored as JSON.

FieldTypeDescription
idUUIDUnique identifier
contentStringMarkdown text content
parent_idUUID?Parent block (null = top-level)
orderf32Sort order among siblings
leveli32Nesting depth
markerenumTodo / Doing / Done / Now / Later / Cancelled
priorityenumA / B / C
propertiesJSONTyped key-value pairs

Pages

A page is a named collection of blocks. Journal pages are date-based (2026-06-11). Regular pages are named (My Project).

Properties

Properties are typed key-value pairs on blocks. Types include: text, number, date, url, checkbox, node. Properties are stored as JSON in the blocks table and indexed via VIRTUAL generated columns.

Annotations

Unified comment system with scope:: block (whole-block comment) and scope:: inline (text-range annotation). Threaded replies via parent_annotation_id. Status lifecycle: pending → in_progress → resolved | dismissed.

References

Bidirectional block references. [[Page Name]] creates page refs, ((block-id)) creates block refs. The refs table maintains bidirectional links with a reverse index.

Intent Search

Heuristic natural-language-to-DSL transformer (no LLM). Type queries like "open tasks by project" and Quilt generates DSL queries. 7 text rules + 4 graph-aware rules. Always shows the generated DSL for editing.

8. Graph Engine

Quilt uses petgraph 0.8.3 for all graph algorithms. The graph is built lazily from block references and analyzed on demand.

Graph Analysis Pipeline

Blocks + Refs ──▶ build DiGraph ──▶ Algorithm ──▶ MCP Tool Response
                       │
                       ├── dijkstra ──▶ shortest_path
                       ├── tarjan_scc ──▶ components
                       ├── eigenvector ──▶ centrality
                       ├── bridges ──▶ critical connections
                       ├── articulation_points ──▶ critical nodes
                       ├── maximal_cliques ──▶ dense clusters
                       └── dinics ──▶ max flow

Example: Find Critical Blocks

# MCP call
quilt_graph_articulation_points()

# Response
{
  "articulation_count": 3,
  "block_ids": ["uuid-1", "uuid-2", "uuid-3"]
}
# These 3 blocks, if removed, would disconnect the knowledge graph.

Example: Dense Concept Clusters

# MCP call
quilt_graph_cliques(min_size: 3)

# Response
{
  "clique_count": 2,
  "cliques": [
    { "size": 4, "members": ["a", "b", "c", "d"] },
    { "size": 3, "members": ["e", "f", "g"] }
  ]
}
